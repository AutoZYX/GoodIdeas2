# Lucky's Playground

> Good Ideas 2.0 的子项目——一个爸爸和 4 岁半的儿子，用 Claude Code 一起做什么

## 背景

Lucky 2021 年 8 月出生，现在 4 岁半。他喜欢的东西非常具体：

- 🦕 **恐龙**——霸王龙、三角龙、剑龙，能准确叫出名字
- 🚜 **工程车**——挖掘机、推土机、自卸车、吊车
- 🧱 **积木**——搭、拆、分颜色、堆高
- 🚓 **警车**——警灯鸣笛对他有近乎魔法的吸引力
- 🏎️ **Monster Truck**——大轮子、撞击感、越野

过去写一个小游戏要一周。现在借 Claude Code，写一个"恐龙点名器"只要一小时。
这意味着——Lucky 的每一个兴趣，都可以变成一个我们父子共创的小项目。

这个子页面就是这些项目的清单和起点。

## 三条原则

**1. Lucky 当 Creator，不是 Consumer**
不是"爸爸做一个游戏给 Lucky 玩"，而是"Lucky 说要什么，爸爸+AI 实现"。
每个项目都要有他亲自决定的部分——选恐龙、定颜色、起名字、录语音。

**2. 4 岁娃的交互底线**
他不识字。UI 必须靠颜色/图标/声音/动作传递信息。
触控优先（iPad 比鼠标更自然），音效必须有。
一个功能超过两步操作，他就放弃了。

**3. 单次 ≤15 分钟**
爸爸有角膜移植史，娃也不该长时间盯屏幕。
宁可做 10 个 5 分钟的小玩意，不做 1 个 1 小时的大项目。
长期项目（图鉴/日记类）按天累积，单次仍是 5 分钟。

## 点子库（按 Lucky 的兴趣分类）

每个项目标注：`Claude Code 要做什么` · `L 等级（参考 Good Ideas 2.0）` · `单次时长` · `起步难度`

### 🦕 恐龙篇

#### D1. 恐龙点名器 / Dinosaur Roll Call
一页网页，10 只恐龙全屏大图轮播。点击任何一只 → 播放咆哮声 + 念出中英文名 + 显示身长体重。
- **Claude Code**：静态 HTML + Web Audio API，咆哮声从 freesound.org 拉
- **L 等级**：L4（Lucky 选要加哪只恐龙，爸爸负责放进去）
- **单次时长**：5 分钟
- **起步难度**：⭐（1 小时）

#### D2. 恐龙骨头拼图 / Dino Skeleton Puzzle
SVG 恐龙骨架拆成 5-6 块，Lucky 拖拽回位。成功 → 骨架"活过来"走两步 + "太棒啦！"
- **Claude Code**：SVG + HTML5 drag-and-drop（或 Konva.js）
- **L 等级**：L4（Lucky 选恐龙，难度爸爸调）
- **单次时长**：10 分钟
- **起步难度**：⭐⭐（半天）

#### D3. 恐龙博士问答 / Ask the Dino Professor
按住麦克风说话（"霸王龙吃什么？"）→ Web Speech API 转文字 → Claude API 用 4 岁娃听得懂的中文回答 → TTS 念出来 + 配图。
- **Claude Code**：前端 + Claude API 封装 + 安全护栏（system prompt："你是一个爱讲恐龙故事的老爷爷，回答要像讲故事，不用术语"）
- **L 等级**：L3（爸爸设 prompt，Lucky 自由问）
- **单次时长**：15 分钟（高注意力消耗）
- **起步难度**：⭐⭐⭐（1-2 晚上）

#### D4. 我画的恐龙会动 / My Drawn Dino Comes Alive
Lucky 在纸上画恐龙 → 拍照 → AI 识别轮廓 → 生成简单的走路动画（头摆动+腿交替）。
- **Claude Code**：图片上传 + Claude Vision + CSS 关键帧动画
- **L 等级**：L4（娃负责画，AI 让它动起来）
- **单次时长**：创作 15 分钟 + 生成 2 分钟
- **起步难度**：⭐⭐⭐（1 天）

---

### 🚜 工程车 & Monster Truck 篇

#### E1. 工地模拟器 / Construction Site
Canvas 游戏：挖掘机挖土坑、推土机推沙堆、自卸车装运。方向键/触屏操控。
- **Claude Code**：Phaser.js 或原生 Canvas + 简单物理
- **L 等级**：L3（物理规则爸爸写，关卡 Lucky 设："我要一座超高的山！"）
- **单次时长**：10-15 分钟
- **起步难度**：⭐⭐⭐（1 天）

#### E2. Monster Truck 撞瓶子 / Monster Truck Crash
Matter.js 物理引擎。一辆 monster truck 从左边冲向一堆罐子/箱子。Lucky 按空格让它跳跃。
- **Claude Code**：Matter.js + 贴图
- **L 等级**：L4（爸爸写物理，Lucky 画涂装 → 贴上去）
- **单次时长**：5-10 分钟
- **起步难度**：⭐⭐（半天）

#### E3. 卡车图鉴卡片 / Truck Trading Cards
Lucky 画一辆 monster truck → 拍照 → AI 识别 + 自动生成数字卡片（名字、最大速度、特技）。爸爸用家里打印机印出来做实体卡牌。
- **Claude Code**：图片上传 + Claude Vision + SVG 卡片模板 + 打印友好布局
- **L 等级**：L4（创作全归 Lucky，AI 配属性）
- **单次时长**：画画 20 分钟 + 生成 2 分钟
- **起步难度**：⭐⭐⭐（1 天）

#### E4. 今天街上的工程车 / Construction Truck Spotter
英国街上工程车很多。出门看到一辆就拍一张，回家上传，AI 识别类型 + 每日累计。做成一面"我今天认识的工程车"墙。
- **Claude Code**：手机上传 + Vision 识别 + 日期归档 + 静态网页生成
- **L 等级**：L3（Lucky 找/拍，爸爸帮上传）
- **单次时长**：每张 2 分钟
- **起步难度**：⭐⭐（半天），长期项目

### 🧱 积木篇

#### B1. 颜色分拣游戏 / Color Sorter
屏幕上方掉下彩色积木，底部有 4 个颜色桶。Lucky 拖到对应颜色里。
- **Claude Code**：纯前端 Canvas，掉落动画+拖拽
- **L 等级**：L5（Lucky 独立玩）
- **单次时长**：5 分钟
- **起步难度**：⭐（1-2 小时）

#### B2. 照着搭塔 / Copy the Tower
展示一张积木塔图片，Lucky 要在屏幕上按相同的顺序堆砌。顺序对 → 塔亮起来。
- **Claude Code**：积木库 + 点击堆叠 + 序列验证
- **L 等级**：L4（爸爸生成关卡，Lucky 解）
- **单次时长**：10 分钟
- **起步难度**：⭐⭐（半天）

#### B3. 积木扫描器 / Block Scanner（线下桥接）
手机摄像头对着 Lucky 搭的真实积木结构 → 屏幕上实时"复现"一个 3D 版本。
- **Claude Code**：WebRTC 摄像头 + 颜色分割 + 简单 3D 渲染（Three.js）
- **L 等级**：L2（需要爸爸帮调光线和距离）
- **单次时长**：设置 10 分钟 + 玩 15 分钟
- **起步难度**：⭐⭐⭐⭐（2 天，有挑战）

---

### 🚓 警车 & 安全意识篇

#### P1. 警灯鸣笛器 / Police Light & Siren
最简单的项目。点屏幕——警灯闪烁（CSS 动画）+ 播放警笛声。可选英国/美国/中国 3 种不同警车造型和声音。
- **Claude Code**：CSS 动画 + 3 个音频文件
- **L 等级**：L5（娃自己玩疯了）
- **单次时长**：3 分钟（但他能重复 10 次）
- **起步难度**：⭐（30 分钟）**← 推荐第一个做的**

#### P2. 警察叔叔，我需要帮忙 / Officer, I Need Help
安全教育项目。AI 扮演警察叔叔，用中英双语问 Lucky：你在哪里？发生了什么？你爸爸妈妈叫什么名字？——模拟真实求助对话。
- **Claude Code**：Claude API + 语音交互 + 英中切换
- **L 等级**：L2（安全教育，爸爸全程陪）
- **单次时长**：10 分钟，每月 1-2 次
- **起步难度**：⭐⭐⭐（1 晚上，但 prompt 要精心设计）

### 🎨 综合创作（Lucky 作 Creator 的长期项目）

#### C1. Lucky 的恐龙图鉴（日更）
一个静态博客。Lucky 每天选一只恐龙，录 30 秒语音（"这是三角龙！它有三个角！"）→ Whisper 转写 → AI 配图生成卡片 → 累计成一本他自己的图鉴。
- **Claude Code**：静态站点（Astro/11ty）+ Whisper API + 图片生成
- **L 等级**：L3（Lucky 负责内容，爸爸负责归档）
- **单次时长**：5 分钟（录音+审）
- **起步难度**：⭐⭐⭐（2 天搭基础设施）
- **输出**：可公开的 Lucky 原创网站

#### C2. Lucky 的工地日记（日更）
和 C1 共享基础设施。每次看到真实工程车，拍照 + Lucky 口述一句话 → AI 整理成日记条目。
- **Claude Code**：C1 的扩展（新增照片上传+日期分组）
- **L 等级**：L3
- **单次时长**：2 分钟
- **起步难度**：⭐（C1 建好后只要 1 小时）

#### C3. 给妈妈的声音明信片 / Voice Postcard to Mom
妈妈在中国。Lucky 录一段中文 → AI 自动加上工程车/恐龙音效作背景 → 生成可分享的小网页 → 发给 Helen。
- **Claude Code**：Web Audio API 混音 + 静态页面 + 分享链接
- **L 等级**：L4（内容和音效都 Lucky 选）
- **单次时长**：5 分钟
- **起步难度**：⭐⭐（半天）
- **情感价值**：对应 Good Ideas 2.0 Ch15 后记的亲情素材

## 推荐起步顺序

从 ROI 最高、单次最快的开始，让 Lucky 爱上"和爸爸一起做东西"：

| 顺序 | 项目 | 为什么先做 |
|-----|------|----------|
| 1 | **P1 警灯鸣笛器** | 30 分钟就能看到效果，Lucky 的"aha 时刻" |
| 2 | **D1 恐龙点名器** | 1 小时，验证"我们一起能做更多" |
| 3 | **B1 颜色分拣** | Lucky 第一次当 Creator（选颜色、难度） |
| 4 | **E2 Monster Truck 撞瓶子** | 引入物理感，他会狂笑 |
| 5 | **C1 恐龙图鉴**基础设施 | 进入长期项目模式 |
| 6+ | 根据 Lucky 当时的兴趣选 | 让他告诉你下一个做什么 |

## 和 Good Ideas 2.0 的关系

Lucky's Playground 不是 Good Ideas 2.0 的附庸，而是它的**活样本**：

- 书里写的**L0-L5 介入等级**，这里每个项目都实际标出来
- 书里写的**"AI 是第三个学习伙伴"**，这里是真实发生，不是理论
- 书里写的**"场景驱动育儿"**，这里每个项目都源自 Lucky 自己说出口的兴趣

有些项目最后会反哺书稿：
- **C3 声音明信片** → Ch15 后记（2026-07 离开英国）的原始素材
- **C1 恐龙图鉴** → Ch14（开源育儿：让 AI 帮你写你自己的"好主意"）的案例
- **D3 恐龙博士** → Ch3（AI 第三个学习伙伴）的活例子
